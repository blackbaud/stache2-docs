The stache-image component is deprecated in Stache version 2.18.0 and higher, and is unavailable starting in Stache version 3.0.0. For Stache SPAs, learn more below about the change.

The image component provides a responsive container to display an image. To display images on your SPA, add them to the src/assets directory in the source folder.

  • src
    • assets
      • image-name.jpg

If your site requires a lot of image files, we recommend you organize the images into subfolders by page name.

After the images are in place, you can reference them in your {{ stache.jsonData.global.productNameShort }} pages. You must reference every image with the ~/assets​/ prefix in order for SKY UX to register it as a static asset and render it.

Image properties

  • imageSource — The path to the image to render.
  • imageAlt — The alt tag to describe the image for accessibility. To meet accessibility requirements, this field should be filled in. For more information on image accessibility reference the W3 docs.
  • caption — (Optional) Adds a caption below the image.
  • captionType — (Optional) Specifies a style for the image caption and divider bar. The valid options are default, success, info, warning, and danger. (Default: default)
  • showBorder — (Optional) Boolean option for displaying a border around the image. (Default: false);
Stache SPAs

To enable more flexibility with the image component, we contributed it to SKY UX as a standalone module. This requires only a few small changes for Stache SPAs.

  • Update your version of Stache to 3.0.0 or higher. Learn how with the Update Stache documentation.
Non-Stache SPAs

If your SPA uses Stache version 3.0.0 or higher, this module is installed and added for you. However, non-Stache users can use this module without Stache. Learn how to set up this module as a dependency in your SKY UX SPA project.

  1. Install the package and plugin as part of your SPAs dependencies.

    npm install @blackbaud/skyux-lib-media --save

  2. Add the module to your app-extras.module.ts file.

    import { SkyImageModule } from '@blackbaud/skyux-lib-media'; @NgModule({ imports: [ SkyImageModule ], exports: [ SkyImageModule ] }) export class AppExtrasModule { }
