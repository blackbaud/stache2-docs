The SKY UX image component provides a responsive container to display an image. To display images in your SPA, add them to the src/assets directory in the source folder.

  • src
    • assets
      • image-name.jpg

After the images are in place, you can reference them in your {{ stache.jsonData.global.productNameShort }} pages. You must reference every image with the ~/assets​/ prefix in order for SKY UX to register it as a static asset and render it. Note that file names and extensions are case-sensitive.

If your site requires a lot of image files, we recommend you organize the images into subfolders by page name.

Image properties

  • caption — (Optional) Adds a caption below the image.
  • captionType — (Optional) Specifies a style for the caption and divider bar. The valid options are default, success, info, warning, and danger. (Default: default)
  • imageAlt — The alt tag to describe the image for accessibility. To meet accessibility requirements, this field should be filled in. For more information on image accessibility reference the W3 docs.
  • imageSource — The path to the image to render.
  • showBorder — (Optional) Boolean option for displaying a border around the image. (Default: false);
Installation

SPAs can use this as a standalone module by setting it up as a dependency.

Note: Most Stache SPAs already have this module installed for you. To confirm, make sure @blackbaud/skyux-lib-media is listed in the package.json file.

  1. To install the package and plugin as part of your SPA's dependencies run the following command from your SPA's root directory.

    npm install @blackbaud/skyux-lib-media --save --save-exact

  2. Add the module to your app-extras.module.ts file.

    import { SkyImageModule } from '@blackbaud/skyux-lib-media'; @NgModule({ imports: [ SkyImageModule ], exports: [ SkyImageModule ] }) export class AppExtrasModule { }
