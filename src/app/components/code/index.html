The stache-code component was contributed to SKY UX as a standalone module and is deprecated in Stache version 3.0.0 and higher. To automatically update your site to use the new component, update Stache to version 3.0.0 or higher. Learn how with the Update Stache documentation.

The code component allows you to format text as a code snippet within a larger block of text. Within the block of text, you wrap sky-code tags around the code snippet. To format longer code snippets that stand alone as separate blocks of text, use the code block component instead.

Code properties

  • escapeCharacters â€” (Optional) Indicates whether to escape special characters such as { and \. Escaping these characters renders them as text instead of code. Defaults to false.

    If a code snippet has text within < and > characters and you set escapeCharacters="true", then the component automatically appends a closing tag. To prevent this, you must use "&lt;" to escape < and "&gt;" to escape >. For example, you must escape the < and > characters in the code snippet public Task<Generic> MethodName() or else the component automatically appends </Generic> to the end: public Task<Generic> MethodName()</Generic>.

Installation

SPAs can use this as a standalone module by setting it up as a dependency.

Note: Most Stache SPAs already have this module installed for you. To confirm, make sure @blackbaud/skyux-lib-code-block is listed in the package.json file.

  1. To install the package and plugin as part of your SPA's dependencies run the following command from your SPA's root directory.

    npm install @blackbaud/skyux-lib-code-block --save --save-exact npm install @blackbaud/skyux-builder-plugin-code-block --save --save-exact

  2. Add the module to your app-extras.module.ts file.

    import { SkyCodeModule } from '@blackbaud/skyux-lib-code-block'; @NgModule({ exports: [ SkyCodeModule ] }) export class AppExtrasModule { }

  3. Add the plugin to your skyuxconfig.json file.

    To use the code in markdown components, Stache include files, or global/json data, you must load the skyux-builder-plugin-code-block plugin after the skyux-builder-plugin-stache plugin. { "name": "stache", "mode": "easy", "compileMode": "aot", "plugins": [ "@blackbaud/skyux-builder-plugin-stache", "@blackbaud/skyux-builder-plugin-code-block" ], }
