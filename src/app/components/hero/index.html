The stache-hero component was contributed to SKY UX as a standalone module and is deprecated in Stache version 2.18.0 and higher. To automatically update your site to use the new component, update Stache to version 3.0.0 or higher. Learn how with the Update Stache documentation.

The hero component adds a hero with a background image, title, and subheading information to your {{ stache.jsonData.global.productNameShort }} page.

Note: Use this component outside of the stache tag.

Hero properties

  • overlayOpacity — Decimal value for how much your image should be darkened. The default is 0.4, acceptable values are any decimal between 0 and 1.
  • backgroundImageUrl — Set this to the name of your image, prepended with ~/assets​/, e.g.: ~/assets​/foo.jpg. All images and other static assets should be stored in the src/assets directory.
    • src
      • assets
        • hero.jpg

Hero heading properties

  • heroTextColor — (Optional) Specify a hex color code as a string to change the hero heading color. (Default: '#fff');

Hero subheading properties

  • heroTextColor — (Optional) Specify a hex color code as a string to change the hero subheading color. (Default: '#fff');
Installation

SPAs can use this as a standalone module by setting it up as a dependency.

Note: Most Stache SPAs already have this module installed for you. To confirm, make sure @blackbaud/skyux-lib-media is listed in the package.json file.

  1. To install the package and plugin as part of your SPA's dependencies run the following command from your SPA's root directory.

    npm install @blackbaud/skyux-lib-media --save --save-exact

  2. Add the module to your app-extras.module.ts file.

    import { SkyHeroModule } from '@blackbaud/skyux-lib-media'; @NgModule({ imports: [ SkyHeroModule ], exports: [ SkyHeroModule ] }) export class AppExtrasModule { }
