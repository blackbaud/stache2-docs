The stache-hero component is deprecated in Stache version 2.18.0 and higher, and is unavailable starting in Stache version 3.0.0. For Stache SPAs, learn more below about the change.

The hero component adds a hero with a background image, title, and subheading information to your {{ stache.jsonData.global.productNameShort }} page.

You need to use this component outside of the stache tag.

Hero properties

  • overlayOpacity — Decimal value for how much your image should be darkened. The default is 0.4, acceptable values are any decimal between 0 and 1.
  • backgroundImageUrl — Set this to the name of your image, prepended with ~/assets​/, e.g.: ~/assets​/foo.jpg. All images and other static assets should be stored in the src/assets directory.
    • src
      • assets
        • hero.jpg
Stache SPAs

To enable more flexibility with the hero component, we contributed it to SKY UX as a standalone module. This requires only a few small changes for Stache SPAs.

  • Update your version of Stache to 3.0.0 or higher. Learn how with the Update Stache documentation.
  • Update your stache-hero, stache-hero-heading and stache-hero-subheading components to sky-hero sky-hero-heading and sky-hero-subheading components.
Non-Stache SPAs

If your SPA uses Stache version 3.0.0 or higher, this module is installed and added for you. However, non-Stache users can use this module without Stache. Learn how to set up this module as a dependency in your SKY UX SPA project.

  1. Install the package and plugin as part of your SPAs dependencies.

    npm install @blackbaud/skyux-lib-media --save

  2. Add the module to your app-extras.module.ts file.

    import { SkyHeroModule } from '@blackbaud/skyux-lib-media'; @NgModule({ imports: [ SkyHeroModule ], exports: [ SkyHeroModule ] }) export class AppExtrasModule { }
