The Restricted View Library is for internal Blackbaud use only. It facilitates a more targeted user experience in Blackbaud documentation by only displaying content that applies to users.

The Restricted View Library provides a component and directive that configure content to make it visible to internal Blackbaud users only and to hide it from all other users.

The Restricted View Library is designed to effectively target content for Blackbaud and non-Blackbaud audiences. It is not a secure way to hide content and should not be used to hide sensitive information.

Library items

  • sky-restricted-view — The sky-restricted-view component wraps restricted content. Within the elements that it it wraps, the component hides all content from users unless they are Blackbaud users who logged in with Blackbaud ID.

  • *skyRestrictedView — The *skyRestrictedView directive is placed as an attribute on a restricted element. It hides any content within that element unless users are Blackbaud users who logged in with Blackbaud ID.

  • sky-restricted-content-alert — The sky-restricted-content-alert component displays an alert that instructs Blackbaud users to log in to see information that is available only to Blackbaud employees. The alert only appears when Blackbaud users who previously viewed the content return but are no longer logged in with Blackbaud ID.

Installation

To use exports from the Restricted View Library library, you must install SkyRestrictedViewModule as a dependency to your project. In the command prompt, run the following command from your SPA's root directory:

npm install --save --save-exact @blackbaud/skyux-lib-restricted-view

After installation, you need to make any exports from SkyRestrictedViewModule available to your SPA by adding the module to the exports section of your SPA's main module. If you are using the SKY UX SDK, add SkyRestrictedViewModule as an export in your app-sky.module.ts file.
