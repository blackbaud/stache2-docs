The Restricted View Library makes content visible to internal Blackbaud users and hides it from other users. This library is for internal Blackbaud use only.

The Restricted View Library includes a component and directive that both allow you to configure content. The library enables you to create a more targeted user experience in Blackbaud documentation by only showing content that applies to users.

Library items

  • sky-restricted-view — The sky-restricted-view component wraps restricted content. The component hides any content within the tags from users who are not logged in with BBID.

  • skyRestrictedView — The skyRestrictedView directive is placed as an attribute on restricted elements. The directive hides any content within the elements from users who are not logged in with BBID.

Installation

To use exports from the Restricted View Library library, you must install SkyRestrictedViewModule as a dependency to your project. In the command prompt, run the following command from your SPA's root directory:

npm install --save --save-exact @blackbaud/skyux-lib-restricted-view

After installation, you need to make any exports from SkyRestrictedViewModule available to your SPA by adding the module to the exports section of your SPA's main module. If you are using the SKY UX SDK, add SkyRestrictedViewModule as an export in your app-sky.module.ts file.
