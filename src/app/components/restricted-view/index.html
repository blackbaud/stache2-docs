The Restricted View Library makes content visible to internal Blackbaud users and hides it from other users. This library is for internal Blackbaud use only.

The Restricted View Library includes both a component and directive allows to configure content. The library enables you to to create a more targeted user experience in Blackbaud documentation by only showing content that applies to a user.

Installation

To use the Restricted View Library library, you must install SkyRestrictedViewModule as a dependency to your project. In the command prompt, run the following command from your SPA's root directory:

npm install @blackbaud/skyux-lib-restricted-view --save --save-exact

After installation, you need to make any exports from SkyRestrictedViewModule available to your SPA by adding the module to the exports section of your SPA's main module. If you are using the SKY UX SDK, add SkyRestrictedViewModule as an export in your app-sky.module.ts file.

Library items

  • sky-restricted-view — The sky-restricted-view component wraps restricted content and hides it. The component hides any content within the tags from users who are nto logged in with BBID.

  • skyRestrictedView — The skyRestrictedView directive is placed as an attribute on restricted elements and hides the content within those elements. The directive hides any content within the elements from users who are nto logged in with BBID.
