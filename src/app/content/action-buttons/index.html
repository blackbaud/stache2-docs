The Action Buttons component provides a clean, visual navigation pattern for linking to multiple documents using the SKY UX Action Button.

Action button properties

  • showSearch — (Optional) Specifies whether to include the search field for the action buttons. Default is true.

  • routes — Specifies the properties for each action button in the list. Each button has the following properties:

    • name — The name of the button.
    • path — The path the button will route to when clicked.
    • icon — The font awesome icon for the button.
    • summary — A description for the button.
Demo Code

When you use this on your site, you'll change the uniquefilename part to the name of your file.

To provide the names, icons, links, and descriptions for the action buttons, you need to create a new data file. Place the file in the src/stache/data folder.

  • src
    • stache
      • data
        • uniquefilename.json

Then, copy and paste the following code into the file and update with your action button information.

[ { "name": "Rock", "path": "/", "icon": "hand-rock-o", "summary": "Enter a description of what to find at the navigation item. " }, { "name": "Paper", "path": "/", "icon": "hand-paper-o", "summary": "Keep the length relatively short." }, { "name": "Scissors", "path": "/", "icon": "hand-scissors-o", "summary": "Write the description with a positive verb about what the user can do on the next page. " } ] Tip: Provide a comma between each item. However, do not use a comma after the last item's }.
