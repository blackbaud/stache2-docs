Update Stache to the newest release The following steps will walk you through updating the various parts of your app needed to use the latest version of Stache. Update Stache version

Open the package.json file found in your app. Here you will need to update the dependencies to install Stache at the latest version.

You can find the latest released version of Stache by checking the change log "dependencies": { "@blackbaud/stache": "2.0.0-beta.3", ... Update the required plugins

Open the skyuxconfig.json file in your project. Be sure you are listing all the required plugins for your project.

NOTE: With Stache Beta release 2.0.0-beta.3, all plugins required by Stache are bundled into the one skyux-builder-plugin-stache plugin. { "name": "stache", "mode": "easy", "compileMode": "aot", "plugins": [ "@blackbaud/skyux-builder-plugin-stache" ], "app": { "title": "Stache 2 Documentation" } } For more information on which plugins are currently required or for optional plugins you may be interested in adding, reference our plugins section. Update App-Extras

Open the src/app-extras.module.ts file. Here we need to ensure we are importing the StacheModule.

NOTE: With Stache Beta release 2.0.0-beta.3, we no longer need to require or declare the StacheConfigService in our providers array. The only Stache relevant code for this file should be the StacheModule. import { NgModule } from '@angular/core'; import { StacheModule } from '@blackbaud/stache'; // Specify entry components, module-level providers, etc. here. @NgModule({ imports: [ StacheModule ], exports: [ StacheModule ], providers: [], entryComponents: [] }) export class AppExtrasModule { } Update node_modules
  1. Delete your node_modules/@blackbaud/ folder. This will remove any outdated versions of plugins or Stache you may have.
  2. From your terminal run npm install. This will install the new version of the Stache module and it's dependencies.
When updating Stache versions, be sure you read through the change log for notes on any breaking changes that may require you to update how certain components are being used. Update components

As with any major update, there is a chance existing components or implementations may have changed. Please read through the change log for notes on any breaking changes that may require you to update how certain components are being used.
