Update Stache to the newest release This tutorial provides step-by-step instructions to update your app to the latest version of Stache. Update the Stache version

Open the package.json file in your project. Update @blackbaud/stache under the dependencies property to install the latest version of Stache.

To find the latest version, check the Stache changelog. "dependencies": { "@blackbaud/stache": "2.0.0-beta.3", ... Before you update, review the changelog for breaking changes. Update the required plugins

Open the skyuxconfig.json file in your project. Be sure you are listing all the required plugins for your project.

NOTE: With Stache Beta release 2.0.0-beta.3, all plugins required by Stache are bundled into the one skyux-builder-plugin-stache plugin. { "name": "stache", "mode": "easy", "compileMode": "aot", "plugins": [ "@blackbaud/skyux-builder-plugin-stache" ], "app": { "title": "Stache 2 Documentation" } } Update App-Extras

Open the src/app-extras.module.ts file. Here we need to ensure we are importing the StacheModule.

NOTE: With Stache Beta release 2.0.0-beta.3, we no longer need to require or declare the StacheConfigService in our providers array. The only Stache relevant code for this file should be the StacheModule. import { NgModule } from '@angular/core'; import { StacheModule } from '@blackbaud/stache'; // Specify entry components, module-level providers, etc. here. @NgModule({ imports: [ StacheModule ], exports: [ StacheModule ], providers: [], entryComponents: [] }) export class AppExtrasModule { } Update node_modules
  1. Delete the node_modules/@blackbaud/ folder for your project to remove any outdated versions of Stache or its plugins.
  2. From the command prompt, run npm install to install Stache and its dependencies.
Update components

As with any major update, existing components or implementations may change. Please make sure to review the Stache changelog for notes on breaking changes that may require you changes in how you use Stache components.
