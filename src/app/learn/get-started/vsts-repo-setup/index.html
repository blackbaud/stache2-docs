Branch Policies and Security in VSTS New Stache SPAs in VSTS are set up with standard policies and security. Learn how to edit these policies for your documentation sites. These instructions are for Blackbaud users with VSTS repos only. Edit Branch Policies

When you create a new SKYUX 2 SPA, it creates a new VSTS repo with a locked down master branch. The person who created the repo is automatically included as a required code reviewer. The master branch also requires at least 1 reviewer for each pull request (PR) into master.

If you need to set up a team as reviewers for a SPA, or if you are going to be the only person working in the repo, you may need to edit the branch policies.

  1. From your VSTS repo main page, select the Branches tab.

  2. Hover your cursor over the master branch, and select the ....

  3. From the menu, select Branch policies.
  4. Under Protect this branch, review the settings and make adjustments. For example, if you are the only person setting up a new Stache SPA, you can temporarily disable the "Require a minimum number of reviewers" setting until others join you in the SPA. We do recommend you have at least one other person review your changes, but we understand in some sites, you may be the only person making edits in the beginning.
  5. Next, scroll down to the bottom of the page and under Automatically include code reviewers, make adjustments as necessary. For example, you may want to chnage your own setting from "Required" to "Optional." You may also want to add other individuals or your scrum team.
  6. When done making adjustments, at the top of the page, select Save changes.
Edit Branch Security

By default, all VSTS Git repos are set up with the same security permissions, including enabling all Products Contributors to contribute to repos. If you want to review and modify individual security permissions for your master branch, review the steps below.

  1. From your VSTS repo main page, select the Branches tab.

  2. Hover your cursor over the master branch, and select the ....

  3. From the menu, select Branch security.

  4. If you have permission, modify the security settings for groups and users. For example, you can add additional users who can edit policies and manage permissions.
  5. After modifying permissions, select Save changes.
