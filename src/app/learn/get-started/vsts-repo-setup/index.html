Branch Policies and Security in VSTS Stache SPAs in VSTS are set up with standard policies and security. Learn how to edit these policies for your documentation sites. These instructions are for Blackbaud users with VSTS repos only. Edit branch policies

When Blackbaud developers create a SKYUX SPA, the App Management Portal creates a VSTS repo with a locked-down master branch. The person who creates the repo is automatically included as a required code reviewer. The master branch also requires at least 1 reviewer for each pull request into master.

You can edit the branch policies if necessary. For example, you can set up a team as reviewers or update the policies to indicate that you are the only person working in the repo.

  1. From your VSTS repo, select the Branches tab.

  2. Hover your cursor over the master branch, and select the ellipses button.

  3. From the menu, select Branch policies.
  4. Under Protect this branch, review the settings and adjust them as necessary. For example, you can disable the "Require a minimum number of reviewers" setting if you are the only person working in the repo, although we generally recommend that at least one other person review your changes.
  5. Scroll to the bottom of the page and make adjustments as necessary under Automatically include code reviewers. For example, you can change your setting from "Required" to "Optional." You can also add other individuals or your scrum team.
  6. At the top of the page, select Save changes.
Edit branch security

By default, all Blackbaud VSTS Git repos have the same security permissions, including enabling all Products Contributors to contribute to repos. You can review and modify individual security permissions for your master branch as necessary.

  1. From your VSTS repo, select the Branches tab.

  2. Hover your cursor over the master branch, and select the ellipses button.

  3. From the menu, select Branch security.

  4. If you have permissions, modify the security settings for groups and users. For example, you can give users rights to edit policies and manage permissions.
  5. Select Save changes.
