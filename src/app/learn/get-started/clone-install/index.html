Clone and Set Up a {{ stache.jsonData.global.productNameShort }} SPA

This tutorial provides step-by-step instructions on how to clone and set up an existing Stache SPA on your computer.

Before you begin this tutorial, you need to complete the SKY UX prerequisites. Clone the Git repo from GitHub

If your repo is in VSTS, skip to step 2.

  1. From GitHub, navigate to your repository.

  2. Select the Clone or download button and copy the clone link.

  3. Open a terminal or command prompt from the location where you store your Git repos. Enter git clone and paste in the link you just copied.

  4. Press Enter (Windows) or return (Mac). The repo clones locally.

Continue with step 3.

Clone the Git repo from VSTS

  1. From VSTS Git , select the repository dropdown and then select All repositories. To search for your repo, use the search field. Most Stache SPAs start with skyux-spa.

    If you're not sure about the VSTS repo name, ask the person who set up the Stache SPA for you.

  2. Select the Clone button.

  3. Under Command line, make sure HTTPS is selected and select the copy button.

  4. Open a terminal or command prompt from the location where you store your Git repos. Enter git clone and paste the link you just copied.

  5. Press Enter (Windows) or return (Mac). The repo clones locally.
NPM install

After the repo clones, change your directory to the new repo folder. For example, enter cd skyux-spa-engineering-system-docs and press Enter (Windows) or return (Mac).

To be able to serve locally, you need to install the SKY UX dependencies. In the command prompt, enter npm install and press Enter (Windows) or return (Mac).

This will take a few minutes. After it completes, you can test the new install by serving locally. Enter skyux serve. When it finishes preparing to serve, a browser will open to a local development instance of your Stache SPA.
