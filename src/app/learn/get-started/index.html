Create a New {{ stache.jsonData.global.productNameLong }} Documentation Site

This tutorial provides step-by-step instructions on how to create a new SKY UX Builder single-page application (SPA) with the {{ stache.jsonData.global.productNameLong }} component library.

Tip: Do you already have a SPA? Learn how to add the {{ stache.jsonData.global.productNameShort }} component library.

If you require a GitHub repo, contact us before continuing.

Install SKY UX prerequisites To prepare your machine for your new {{ stache.jsonData.global.productNameShort }} documentation site, you need to install SKY UX's prerequisites.

  1. To load your local single-page application into SKY UX Host, you must install the skyux-ca.crt root certificate. Follow the SSL certification installation instructions on the SKY UX website.

  2. Install Node.js version 6+.

    Do not use Node.js verison 7+.

    Do you already have Node installed? To check your version, run node â€“v from the command line.

    3. For Mac OS X, we recommend that you use Node Version Manager (nvm) to wrap your NodeJS installation so that it installs in your user directory and avoids permissions-related issues.

  3. Install Git.

  4. From the command line, run npm install -g @blackbaud/skyux-cli.

  5. Verify you have access to Visual Studio Team Services (VSTS).

Create and provision your SKY UX application in the App Management Portal

Next, we'll create your SKY UX application and repo with the SKY UX App Management Portal. This creates a Visual Studio Team Services (VSTS) repo for the application and also creates VSTS build and release definitions.

  1. Navigate to the SKY UX App Management Portal.

  2. Enter a name for your {{ stache.jsonData.global.productNameShort }} site. We request that you append any documentation sites with -docs. For example: engineering-system-docs.

    3. Tip: The name you enter here will become part of your site's URL.

  3. Review and approve the terms and conditions.

  4. Select Create Your Application. This process may take a moment.

  5. When the application is created, a ready page appears. We recommend you save a copy of this page.
  6. Copy the Git Repo URL.
Clone your repo and set up your {{ stache.jsonData.global.productNameShort }} site

In this step, we'll set up your local working copy of the VSTS repo and install {{ stache.jsonData.global.productNameShort }}.

  1. Navigate to the folder on your computer where you have your VSTS and GitHub repos, or where you'd like to save your new repo.

  2. Windows only: Right-click and select Git Bash Here.

  3. In the command line, enter skyux new --template stache.

  4. In the first prompt, enter the name you provided in the SKY UX App Management Portal. For example engineering-system-docs. The SKY UX CLI creates a folder with this name and prefixes it with "skyux-spa."

  5. In the second prompt, paste in the URL to your repo that we copied in step 2.6.

    6. Your repo is now being cloned and set up. This will take a few minutes.

Verify authentication for your application

After you create your application, it is available on the public Internet. Be careful to not allow anonymous users access to protected Blackbaud assets, such as user data or internal Blackbaud systems.

We default to Blackbaud Authentication turned on, but we recommend your first step be to verify it.

  1. Navigate to the repo on your local machine.

  2. Open skyuxconfig.json file in your preferred text editor.

  3. Find "auth": true as seen below and double-check that the config setting is true.

View your new {{ stache.jsonData.global.productNameShort }} application

In {{ stache.jsonData.global.productNameLong }}, to see your site is slightly different from {{ stache.jsonData.global.productNameShort }} 1.

From the command line, enter skyux serve. A new browser window opens to your documentation site.

Tip: The URL in your browser is the same as your production URL. The red bar at the top of the page indicates when you are in "dev mode." To see your production site, select the red bar and then select delete next to Custom Config. To return to dev mode, stop serving in your command prompt and serve again.
