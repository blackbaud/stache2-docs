To expand on existing SKY UX Builder functionality and create new functionality, {{ stache.jsonData.global.productNameShort }} uses plugins. Each plugin is geared toward a specific purpose, and most of them are required to make use of certain {{ stache.jsonData.global.productNameShort }} components.

All the plugins that {{ stache.jsonData.global.productNameShort }} requires are bundled into the @blackbaud/skyux-builder-plugin-stache plugin.

If you create a project with the skyux new -t stache command, plugins are added for you. If you upgrade to a newer version of Stache, make sure all required plugins are installed. Required plugin

For Stache sites, the required @blackbaud/skyux-builder-plugin-stache plugin bundles all other plugins that {{ stache.jsonData.global.productNameShort }} requires.

"@blackbaud/skyux-builder-plugin-stache" Optional plugin

Email whitelist plugin

The optional @blackbaud/skyux-builder-plugin-auth-email-whitelist plugin allows you to enable Blackbaud authentication for internal sites. This restricts access to a SKYUX SPA to users with valid Blackbaud email addresses only.

@blackbaud/skyux-builder-plugin-auth-email-whitelist

Add plugins to your project

After you update the package.json file to install the latest version of Stache, you need to add required plugins to the plugins array in the skyuxconfig.json file.
