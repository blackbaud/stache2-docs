Update {{ stache.jsonData.global.productNameShort }} This tutorial provides step-by-step instructions to update your app to the latest version of {{ stache.jsonData.global.productNameShort }}. Update the {{ stache.jsonData.global.productNameShort }} version

Open the package.json file in your project. Update @blackbaud/stache under the dependencies property to install the latest version of {{ stache.jsonData.global.productNameShort }}.

To find the latest version, check the {{ stache.jsonData.global.productNameShort }} changelog. "dependencies": { "@blackbaud/stache": "2.0.0-beta.9", ... Before you update, review the changelog for breaking changes. Update the dev dependencies

Open the package.json file in your project. Update @blackbaud/skyux and @blackbaud/skyux-builder under the dev-dependencies property to install the latest versions of each.

Your updated package.json file with all {{ stache.jsonData.global.productNameShort }} dependencies should look as follows. NOTE: With {{ stache.jsonData.global.productNameShort }} Beta release {{ stache.jsonData.global.stacheVersion }}, codelyzer was removed as a dependency, and skyux was changed to a dev dependency. "dependencies": { "@blackbaud/stache": "2.0.0-beta.9" }, "devDependencies": { "@blackbaud/skyux": "2.0.0-rc.4", "@blackbaud/skyux-builder": "1.0.0-rc.6" } Update the required plugins

Open the skyuxconfig.json file in your project. Be sure you are listing all the required plugins for your project.

NOTE: With {{ stache.jsonData.global.productNameShort }} Beta release {{ stache.jsonData.global.stacheVersion }}, all plugins required by {{ stache.jsonData.global.productNameShort }} are bundled into the one skyux-builder-plugin-stache plugin. { "name": "stache", "mode": "easy", "compileMode": "aot", "plugins": [ "@blackbaud/skyux-builder-plugin-stache" ], "app": { "title": "Stache 2 Documentation" } } Update App-Extras

Open the src/app-extras.module.ts file. Here we need to ensure we are importing the StacheModule.

NOTE: With {{ stache.jsonData.global.productNameShort }} Beta release {{ stache.jsonData.global.stacheVersion }}, we no longer need to require or declare the StacheConfigService in our providers array. The only {{ stache.jsonData.global.productNameLShort }} relevant code for this file should be the StacheModule. import { NgModule } from '@angular/core'; import { StacheModule } from '@blackbaud/stache'; // Specify entry components, module-level providers, etc. here. @NgModule({ imports: [ StacheModule ], exports: [ StacheModule ], providers: [], entryComponents: [] }) export class AppExtrasModule { } Update node_modules
  1. Delete the node_modules/@blackbaud/ folder for your project to remove any outdated versions of {{ stache.jsonData.global.productNameShort }} or its plugins.
  2. From the command prompt, run npm install to install {{ stache.jsonData.global.productNameShort }} and its dependencies.
