As you are getting to know Stache, use the documentation below to understand how it's organized and how to create documentation using Stache. For information about specific components, refer to our Components.

Stache wrapper <stache>

The Stache wrapper is the primary tag in a Stache project, and contains all the other Stache tags and content. Most of your page level configuration is done through the Stache wrapper, including layouts and navigation. Typically, this is the first tag you will want to add to any Stache page.

Stache takes a prescriptive approach to configuration. Most of the properties on the Stache tag have defaults, so you will rarely have to set them yourself. The main exception is the page title.

Stache layouts enable consistent formatting for your content. You specify the layout on the stache tag's layout property. Swapping layouts is as simple as updating the property with a new layout name. There are three different layouts you can use:

The currently active page in the sidebar navigation show any children underneath. All other child pages are hidden until the route becomes active.

The order the sidebar routes appear is determined first by the navOrder attribute on the {{stache.jsonData.global.productNameShort}} tag. If there is no navOrder specified, or if two sibling routes have the same navOrder, the routes will be ordered alphabetically by name.

The blank layout provides no formatting for the content inside. This allows you to do whatever you wish inside of the container but still make use of the utility of the stache tag.

Navigation

Stache provides several navigational utilities for moving users quickly thorough your content. Most of these are enabled by default.

Back To Top

For long form content, having to scroll manually to the top of the page is annoying. The back to top button provides a quick way to navigate back to the top of a page. Back to top is enabled by default. You can disable it on the stache tag.

<stache pageTitle="My Sample Page" showBackToTop="false"> // Your content here. </stache>

Breadcrumbs

Breadcrumbs provide you a direct path from your current location to the home page. This pattern gives users an easy way to see exactly where they are in the context of the site. Breadcrumbs are enabled by default. You can disable them on the stache tag when you don't want them to display, such as on the home page.

<stache pageTitle="My Sample Page" showBreadcrumbs="false"> // Your content here. </stache>

Page Anchor

Page anchors provide inter-page navigation and allow you to link to specific sections of a page. These serve a dual purpose as page section headers. Page anchors are used to generate the links for the table of contents.

<stache pageTitle="My Sample Page"> <stache-page-anchor> Section One </stache-page-anchor> // Some content <stache-page-anchor> Section Two </stache-page-anchor> // More content </stache>

Table of Contents

The table of contents allows you to navigate quickly within a page using page anchors. It also serves as an overview of the page contents. This component, combined with the back to top and page anchors, creates a quick and effortless way to navigate long form content.

Note: This is the only navigation component not enabled by default.

To use the table of contents on your page, enable it on the stache tag.

<stache pageTitle="My Sample Page" showTableOfContents="true"> <stache-page-anchor> Section One </stache-page-anchor> // Some content <stache-page-anchor> Section Two </stache-page-anchor> // More content </stache> Images

JSON Data Service

Stache can consume JSON data files and provide them to you for use in your Stache templates and components. There is a default global.json file that you can use for any variable data you wish to include.

Note: To use the data, you need to add the Stache plugin to the plugins array in skyuxconfig.json. See plugins for more information.

Data you wish to consume should be placed in the src/stache/data/ directory. For example:

  • src
    • stache
      • data
        • example.json
Note: Make sure to name your files with names that can be converted to valid object keys. Ex: data_file.json or dataFile.json

We can then use that data in a template like so:

{{ stache.jsonData.example.key }}

If you want to use the data in a component, you can do so with the StacheJsonDataService by adding it to your component:

import { Component, OnInit } from '@angular/core'; import { StacheJsonDataService } from '@blackbaud/stache'; @Component({ selector: 'my-component', template: './my-component.component.html' }) export class MyComponent implements OnInit { public allData: any; public specificData: any; constructor( private dataService: StacheJsonDataService ) { } ngOnInit() { this.allData = this.dataService.getAll(); this.specificData = this.dataService.getByName('example'); } } Command Line

Stache asks you to use commands when installing and working with your repo.

On a Mac, use Terminal.

On a PC, type cmd in the search box to access the command line. Once you install Git, you can right-click in any folder and select Git Bash Here to access a command line.
