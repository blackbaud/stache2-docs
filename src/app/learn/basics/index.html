As you are getting to know Stache, use the documentation below to understand how it's organized and how to create documentation using Stache. For information about specific components, refer to our Components. Stache Wrapper

The Stache Wrapper is the primary tag in a Stache project, and contains all the other Stache tags and content. Most of your page level configuration is done through the Stache Wrapper, including layouts and navigation. Typically, this is the first tag you will want to add to any Stache page.

Stache takes a prescriptive approach to configuration. Most of the properties on the Stache tag have defaults, so you will rarely have to set them yourself. The main exception is the page title.

Navigation

Stache provides several navigational utilities for moving users quickly thorough your content. Most of these are enabled by default and handled for you automatically.

Back To Top

For long form content, having to scroll manually to the top of the page is annoying. The back to top button provides a quick way to navigate back to the top of a page. Back to top is enabled by default. You can disable it on the stache tag.

Breadcrumbs

Breadcrumbs provide you a direct path from your current location to the home page. This pattern gives users an easy way to see exactly where they are in the context of the site. Breadcrumbs are enabled by default. You can disable them on the stache tag when you don't want them to display, such as on the home page.

Page Anchor

Page anchors provide inter-page navigation and allow you to link to specific sections of a page. These serve a dual purpose as page section headers. Page anchors are used to generate the links for the table of contents.

Table of Contents

The table of contents allows you to navigate quickly within a page using page anchors. It also serves as an overview of the page contents. This component, combined with the back to top and page anchors, creates a quick and effortless way to navigate long form content.

Include

Include enables you to single source your content and reuse it throughout your documentation. This is useful if you have small chunks of content that you find yourself repeating in several places. You can use an include, update it in one place, and it'll change everywhere it's used.

Content you wish to include should be placed in the src/stache/includes/ directory. For example:

  • src
    • stache
      • includes
        • example.html

We can then refer to that file anywhere in our app with the include tag:

<stache-include fileName="example.html"> </stache-include> Note: Includes are for static content. For dynamic content, create a custom component instead.
