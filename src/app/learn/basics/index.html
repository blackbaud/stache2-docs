Stache wrapper

The {{ stache.jsonData.global.productNameShort }} wrapper is the primary element in a {{ stache.jsonData.global.productNameShort }} project. It contains most of the other {{ stache.jsonData.global.productNameShort }} tags and content. Most page-level configuration is handled by the {{ stache.jsonData.global.productNameShort }} wrapper, including layouts and navigation. Typically, this is the first tag to add to any {{ stache.jsonData.global.productNameShort }} page after the app-nav tag.

{{ stache.jsonData.global.productNameShort }} takes a prescriptive approach to configuration. Most properties on the stache tag have defaults, so you rarely need to set them yourself. The main exceptions are the pageTitle, navTitle, and navOrder.

Application navigation

To provide top-level navigation within your app, you can use the app-nav component. This shared component from the SKY UX default template allows you to display a navigation bar at the top of pages.

For internal Blackbaud SPAs, we recommend the omnibar instead of the app-nav component. However, it is for Blackbaud staff use only. To learn how to set up top-level navigation with the omnibar, see step 4 of the Create a Blackbaud Stache SPA tutorial.

Display navigation bar

To display a navigation bar at the top of a page, place the app-nav tags at the start of the page's index.html file. To include the navigation bar throughout your SPA, start every page with the app-nav tags.

<app-nav></app-nav>

Set up navigation bar

To display entries in the navigation bar, you must manually configure the top-level navigation for your SPA. To use localization strings for the top-level navigation items, see the create a page tutorial on the SKY UX website.

  1. In the src/app/shared folder, open the app-nav.component.ts file. You need entries in the AppNavComponent class for each top-level navigation item.

  2. Use the titleKey property to specify a label for a navigation item.

    If you specify a localization string from resources_en_US.json, the navigation bar displays the message value from that localization string.

  3. Use the path property to specify where to direct users when they click a navigation item. When you specify the path to a folder in your SPA, the navigation bar opens the index.html file in that folder.

    4. export class AppNavComponent { public nav = [ { titleKey: 'app_nav_home', path: '/' }, { titleKey: 'app_nav_about', path: '/about' }, { titleKey: 'Demo', path: '/demo' } ]; }
Custom CSS

In {{ stache.jsonData.global.productNameLong }}, you can add custom CSS (or SCSS) through SCSS files. We recommend you create a file named app.scss and place it in the src/app folder. For those who don't know SCSS, you can use CSS inside an SCSS file. For example:

//CSS img { border: 1px solid #cccccc; }

After the app.scss file is in place, you need to tell {{ stache.jsonData.global.productNameLong }} where to find your custom CSS. From the src/app folder, open the app-extras.module.ts file and insert this line of code at the beginning of the file.

require('style-loader!./app.scss'); Command line

Stache asks you to use commands when installing and working with your repo.

On a Mac, use Terminal.

On a PC, type cmd in the search box to access the command line. After you install Git, you can right-click in any folder and select Git Bash Here to access a command line. However, do not use Git Bash when you run skyux new --template {template-url} because it causes a conflict that prevents the install from completing.

VSTS for Blackbaud staff

Unless you are working locally, all new SPAs are set up as Git repos in VSTS . VSTS has pull requests just like GitHub. However, there are three key differences to know about.

  1. When you create a pull request (PR), please remove [Products]\Products Team from the reviewers or edit the branch policies.
    If you have VSTS permission, from your repo's Branches page, hover over the branch row, select the "...", and then select Branch policies. On this page you can set many branch settings, including who to automatically include on code reviews.
    If you don't have permission, speak to your repo creator/owner.
  2. Each time you create a PR, or make a change to your pull request, it kicks off a build. The build takes a few minutes to complete and tests to make sure there aren't any problems with your code before it merges.
  3. The PR build is one part of the PR approval process. Most VSTS repos also require you to get at least one code review approval.

Because of these differences, each change to your repo will take, at minimum, a few minutes to release to your live SPA.

You can use tools such as VS Code to work with a local clone of your Stache site, but the web-based editor in VSTS lets you make quick changes without leaving the browser. To learn how to make changes with the VSTS web-based editor, see the Edit a VSTS Stache 2 Site in Your Browser tutorial .

Tools

To help you work with {{ stache.jsonData.global.productNameLong }} SPAs, we have some tool recommendations.

VS Code Code editor

Visual Studio Code (VS Code) is a code editor well suited to working with SKY UX SPAs. In addition, VS Code enables many extensions that can help you validate your work before checking in your changes. Here are some extensions we recommend:

  • TSLint
  • HTMLHint
  • highlight-matching-tag
  • Guides
  • seti-icons

If you are a developer, you'll want to search the marketplace for technical extensions that can help you work more efficiently.

Markdown Editor Converts markdown to html

If you are new (or rusty) to HTML, but know markdown, this (GitHub-Flavored) Markdown Editor tool was recommended to us. When you navigate to the page, it creates a unique link. You can edit the text in Markdown on the left, and the editor converts it to HTML on the right. Then, you can use the inspect tool in your browser to copy the HTML.

Best of all, you can share this unique link with others so they can edit the content too. This is a great way in general to collaborate on content before posting to your SPA.
