We try to prevent issues by creating exceptional documentation, but we know you may run into problems from time-to-time. On this page, learn how to resolve them. Based on your feedback and trends that we notice, we'll continue to update our documentation and add notes here. Let us know how we can help you better. NPM install

Scenarios

  • When you were running skyux new --template https://github.com/blackbaud/skyux-template-stache.git or npm install, did the install time out?

  • Are you having a problem serving your site locally?

  • Did you update the dependencies in the package.json file?

Solution

You may want to re-run npm install. Before you do that, we recommend you delete your entire node_modules folder. In the command line for the Stache site folder, enter npm cache clear. When that completes, enter npm install.

Page doesn't load locally

Scenario

When you run skyux serve, does a browser window open to your "in-dev" site, but all you see is a bouncing green ball and the page doesn't load?

Solution

When this happens, it means there's an error in the code for your Stache site. We recommend you use the browser developer tools to find the error. For example, in Chrome, right-click on the page and select Inspect. Then, select the Console tab. Do you see a lot of red text? Scroll up to the top to see if you can find a clue to locate the error in the code.

Common issues we've noticed:

  • Did you provide an opening tag, like <stache-page-anchor> but not the closing one like </stache-page-anchor>?
  • Check your JSON files for errors. These can include unnecessary commas, missing commas, unncessary spaces, unncessary tabs, missing characters ( {, [, }, ], etc), and improper formatting.
  • Did you use punctionation, like a -, when you reference json files from a page? For example, when you use the action buttons component, don't use punctuation (ex. dashes) in your unique filename.
  • Did you reference an image that wasn't added to your assets folder?
  • Did you misspell a component? For example, if you reference stace-code instead of stache-code, your SPA will try to find that misspelled component and fail.

If you've made a recent change and everything looks good, you can also stop serving and serve again to see if that fixes the issue. If not, keep looking because there's an error in there somewhere. Still can't find it? Contact us and we'll be glad to help hunt it down.

Formatting issues in VS Code

Scenario

When you paste HTML content into VS Code and use the Markdown component, does it automatically format the content to add indentations that render the page as a code block?

Solution

Edit your VS Code settings for formatting pasted content. Select Files, Preferences, Settings, and in User Settings, add "editor.formatOnPaste": false to the list.
