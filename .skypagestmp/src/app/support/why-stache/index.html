Stache is brought to you by the Blackbaud Strategic Content Development team. We constantly strive to improve documentation for the benefit of all consumers. What is it?

Stache is a component library for SKY UX SPAs, which means all the goodness of SKY UX plus features customized for documentation. With our documentation patterns and components coupled with SKY UX, you have the ability to create a well-designed and consistent documentation experience optimized for your consumer.

However, Stache isn't for early collaborative documentation or projects where a team is taking a lot of notes and working through a solution together.

When you need to:

  • Share knowledge, including code, across a wide group
  • Teach
  • Onboard
  • Raise awareness
  • Provide tooling help content
  • Optimize your content for an external audience

Then, that's the time you need to invest in creating good documentation. Good documentation can result in less friction for your users learning and understanding. When you enable efficient self-service, you can spend less time answering questions.

  • Source control with Git (Do you want the ability to review content before it's live?)
  • With a few simple commands, you can have it set up locally on your computer.
  • Extensible (Because it's a SKY UX app, you can build out custom features you need or even mix tooling and documentation.)
  • Open source

 When to use what

Question #1: Who's going to use your content?

If it's a limited group of people, like a scrum team, then it's up to your group how you want to document team knowledge and processes. Use what's best for you.

Do you have consumers? If yes, we strongly recommend you create a Stache site. We want your information to be successfully delivered and used. Our tools and processes support that effort.

Question #2: Is your site going to be public?

If your site is going to be public-facing (now or in the future) to any and all users on the Internet, then the choice is easy. Use Stache. Stache tools and processes help you create documentation orientated for an open site.

Question #3: Do you primarily need to share files?

If the primary purpose of your content is to only share files, like Microsoft Word documents, then don't create a Stache site. If you're a Blackbaud staff, we recommend you use our Office365 tools (SharePoint) to set up a document library instead.
